# RB-V92-003: Context Budget Overflow Spike
## Trigger
Context overflow rate exceeds 20% for a sustained 5-minute window.

## Detection
Spike in `BREVIO_context_overflow_total` and policy denials `CONTEXT_BUDGET_EXCEEDED`.

## Immediate Actions
1. Enable conservative context profile for impacted tenants.
2. Drop lowest-priority context classes first (non-critical history/tool traces).
3. Capture representative overflow traces for analysis.

## Mitigation
1. Tune context allocator weights and per-tier token caps.
2. Review recent prompt changes that increased context footprint.
3. Adjust retrieval top-k and history windows to reduce token pressure.

## Recovery
1. Roll out optimized budget configuration in staged cohorts.
2. Monitor latency/quality trade-offs during rollback/roll-forward.
3. Publish updated allocator guidance to engineering runbook notes.

## Verification
1. Overflow rate returns below threshold.
2. No increase in guardrail or policy-deny regressions.
3. Interactive latency remains within SLO after changes.

## Escalation
Escalate to Brain/Context owner if overflow remains high after config rollback or if user-facing quality degrades materially.
