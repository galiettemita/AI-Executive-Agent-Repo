# RB-V92-001: RAG Collection Degradation
## Trigger
RAG eval score drops by more than 10% over baseline for an active collection.

## Detection
Alert from `BREVIO_rag_faithfulness_score` / `BREVIO_rag_relevance_score` plus failed `rag_eval_v1` workflow checks.

## Immediate Actions
1. Freeze new ingests for affected collection(s).
2. Pin retrieval traffic to last known-good collection snapshot.
3. Open incident record with affected workspace IDs and collection IDs.

## Mitigation
1. Compare current chunking/reranker config against prior version.
2. Rollback reranker or chunking profile if drift is detected.
3. Run targeted retrieval probes on gold-set queries to confirm quality recovery.

## Recovery
1. Re-enable ingest in canary mode (single workspace).
2. Gradually restore normal traffic after two consecutive healthy eval windows.
3. Document final config and root cause in incident notes.

## Verification
1. `rag_eval_v1` passes faithfulness/relevance thresholds.
2. No sustained drop in retrieval hit quality for canary and production cohorts.
3. No active degradation alerts for 30 minutes.

## Escalation
Escalate to on-call ML/RAG owner if quality remains below threshold after rollback, or if >3 workspaces are impacted.
