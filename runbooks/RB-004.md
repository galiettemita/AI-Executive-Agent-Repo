# RB-004: Provisioning Workflow Failure and Compensation

## Trigger
- `provisioning_v9` workflow step failure or compensation chain failure.
- `BREVIO.provision.server.failed.v1` emitted.

## Detection
- Inspect Temporal history for failed step and idempotency key.
- Verify compensation order is exact reverse of executed steps.

## Immediate Actions
1. Block further provisioning for the affected server/capability.
2. Capture failed artifact digest, signature status, and policy decision input.
3. Confirm compensation steps completed for created resources.

## Mitigation
- Re-run from first failed step only after root cause is resolved.
- If artifact verification fails, quarantine server candidate immediately.
- If OAuth callback failed, force new approval flow with fresh nonce.

## Recovery
- Resume provisioning queue after two consecutive healthy workflow runs.
- Publish operator-visible summary and impacted request IDs.

## Verification
- No orphaned resources in `workspace_mcp_servers`.
- Step logs and compensation events are complete and ordered.

## Escalation
- Escalate to platform and security if compensation cannot complete.
