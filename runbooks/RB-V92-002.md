# RB-V92-002: Tool Auto-Quarantine Response
## Trigger
`BREVIO.tool_health.quarantined.v1` event emitted for a production tool.

## Detection
Tool health scorer marks status as quarantined; increased failures and breaker-open events.

## Immediate Actions
1. Confirm quarantine reason and recent failure window.
2. Route requests to configured fallback path/tool if available.
3. Notify operations channel with workspace and tool key impact.

## Mitigation
1. Inspect upstream provider status and credential validity.
2. Reduce call rate for affected connector to avoid retry storm.
3. Apply temporary operator override only if documented risk is acceptable.

## Recovery
1. Execute controlled health probe sequence (half-open behavior).
2. Lift quarantine when success criteria are met.
3. Backfill failed queued actions where safe/idempotent.

## Verification
1. Tool health score returns to healthy/degraded (not quarantined).
2. Error rate below configured threshold for 15 minutes.
3. Fallback traffic returns to baseline.

## Escalation
Escalate to connector owner and incident commander if quarantine persists >30 minutes or affects critical workflows.
