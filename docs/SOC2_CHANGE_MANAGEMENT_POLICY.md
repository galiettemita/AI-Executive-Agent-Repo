# SOC 2 Change Management Policy

## Purpose
Ensure production changes are reviewed, tested, and traceable.

## Change Types
- Standard: low risk, pre-approved changes
- Normal: requires review and testing
- Emergency: urgent changes with post-approval review

## Requirements
- All code changes require a PR, code review, and CI checks.
- Production changes must include a rollback plan.
- Emergency changes must be documented and reviewed within 24 hours.

## Logging
- Maintain a change log with PR links, approvals, and deploy metadata.

## Reviews
- Quarterly review of change management compliance.
