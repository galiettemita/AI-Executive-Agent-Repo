5. Temporal Workflow Gaps
The blueprint mandates exclusive use of Temporal for all async operations. No goroutine, cron, or background thread may replace a Temporal workflow. Workflows must be deterministic and replay-safe — no nondeterministic operations (time, random, I/O) inside workflow functions.

7. OpenClaw Skills Integration Gaps

7.1 Critical Skills Infrastructure

9.3 Dead Letter Queues

10. Testing & Evaluation Gaps
Blueprint mandates a dual test system: deterministic software tests for system behaviour + eval-driven tests for model variability. Both must pass before production deploy.


10.1 Critical Test Scenarios
Scenario 1 — Message Processing Happy Path: WhatsApp webhook → Gateway normalize → Brain classify (confidence ≥0.7) → Brain decompose (valid DAG) → Hands execute → response within SLO.
Scenario 2 — Parallel Skill Execution: 'Order groceries and play music' → 2 parallel tasks → total latency = max(task1, task2), NOT sum.
Scenario 3 — Circuit Breaker: 5 consecutive failures → OPEN → immediate CIRCUIT_OPEN error → half-open probe → close after success.
Scenario 4 — Budget Exhaustion: Budget set to $0.05 → messages until exhausted → graceful degradation to keyword-only → user notification.
Scenario 5 — Trust Promotion: trust_score ≥ 0.85 AND success_count_30d ≥ 20 AND failure_count_30d = 0 for 14 days → promotion proposal → operator notification.

11. Security & Compliance Gaps
11.1 Cryptographic Controls
Approval tokens: HMAC-SHA256 signed. CRITICAL TTL=30s, ELEVATED TTL=5min. Nonce-unique. key_versions table authoritative.
OAuth token storage: AES-256-GCM envelope: {ciphertext, nonce, key_version, encrypted_at}. Key material from Secrets Manager.
Key rotation: Dual-key read window ≥10 min. Background re-encryption job. key_versions table.
Webhook signatures: HMAC-SHA256(app_secret, raw_body_bytes). Timing-safe compare. Replay protection: nonce cache TTL=600s.
Audit log integrity: Hash chain HMAC. Append-only enforced by DB triggers. No UPDATE/DELETE. Separate DB role.
Artifact verification: Ed25519 on image digests (sha256). SBOM + vulnerability scan required.

11.2 V9.2 PII Encryption & MCP Sandboxing
pii_encryption_policies: Per-field encryption config with key rotation schedule. Separate Postgres role for pii_read only.
mcp_sandbox_profiles: Per-MCP-server filesystem access rules, network egress allow-list, syscall filter. No MCP server runs outside sandbox.
