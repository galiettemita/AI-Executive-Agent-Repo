# RB-002: Workspace Isolation Alert

## Trigger
- Any evidence of cross-workspace read/write visibility.
- Failed workspace isolation acceptance gate or unexpected RLS bypass.

## Detection
- Review DB audit trail for queries missing `app.workspace_id`.
- Run isolation canary query for two known workspaces and compare row visibility.

## Immediate Actions
1. Set system to `D4` (read-only mode) for impacted services.
2. Disable non-essential write endpoints in control policy.
3. Capture exact query path, service identity, and trace/span ids.

## Mitigation
- Patch middleware to enforce `SET LOCAL app.workspace_id` for every transaction.
- Reject requests lacking workspace context at API edge.
- Backfill incident audit entries with affected table/key references.

## Recovery
- Re-run isolation test suite and contract tests.
- Restore write operations incrementally (gateway -> control -> executor).

## Verification
- Isolation test passes across all workspace-scoped tables.
- No unauthorized rows returned in red-team cross-tenant tests.

## Escalation
- Security incident severity P1 until root cause and remediation are confirmed.
