# RB-V10-002: Outbound Call System Operations

## Scope
Outbound call subsystem: approval gates, provider failover, transcript persistence.

## Architecture
- **Approval Gate**: All calls require approved `call_approval_request` (NNR).
- **Phone Verification**: Google Places API lookup before dialing.
- **Provider Failover**: Automatic at 5% error rate threshold.
- **Transcript Persistence**: Text only; no raw audio stored (NNR).

## Common Operations

### Check Provider Health
```sql
SELECT provider_name, status, health_score, current_concurrent_calls
FROM call_providers WHERE workspace_id = '<WS_ID>';
```

### View Pending Approvals
```sql
SELECT id, status, purpose, expires_at
FROM call_approval_requests
WHERE workspace_id = '<WS_ID>' AND status = 'pending'
ORDER BY created_at DESC;
```

### Approve/Deny via API
```bash
curl -X POST https://executor.brevio.app/v1/executor/call/approve \
  -H "Content-Type: application/json" \
  -d '{
    "approval_id": "<APPROVAL_ID>",
    "decision": "approve",
    "decided_by": "<OPERATOR_ID>",
    "reason": "verified by operator"
  }'
```

### Force Provider Failover
```sql
UPDATE call_providers SET status = 'failover', health_score = 0
WHERE workspace_id = '<WS_ID>' AND provider_name = 'vapi';
```

### Reset Provider After Recovery
```sql
UPDATE call_providers SET status = 'active', health_score = 1.0
WHERE workspace_id = '<WS_ID>' AND provider_name = 'vapi';
```

## Incident: Calls Not Completing

1. Check provider health: `SELECT * FROM call_provider_health_log ORDER BY created_at DESC LIMIT 20;`
2. Check for rate limit exhaustion: `SELECT * FROM call_rate_limits WHERE workspace_id = '<WS_ID>';`
3. Check Temporal workflow status for `OutboundCallWorkflow`.
4. Verify Google Places API key is set: `GOOGLE_PLACES_API_KEY` env var.

## Incident: Blocked Number Complaints

1. Check blocklist: `SELECT * FROM call_number_blocklist WHERE workspace_id = '<WS_ID>';`
2. Remove if incorrectly blocked:
```sql
DELETE FROM call_number_blocklist WHERE workspace_id = '<WS_ID>' AND number_hash = '<HASH>';
```

## Rollback
Call subsystem is stateless from the service perspective. To disable:
1. Set all providers to `offline`: `UPDATE call_providers SET status = 'offline';`
2. This prevents new calls while preserving existing data.
