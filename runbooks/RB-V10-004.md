# RB-V10-004: Brain Ingress and Message Processing

## Scope
Brain service message ingestion, Temporal workflow dispatch, idempotency.

## Architecture
- `POST /v1/brain/ingest` accepts a `MessageEnvelope` JSON payload.
- Uses `msg-{envelope.ID}` as Temporal workflow ID for idempotency.
- Dispatches `MessageProcessingWorkflow` on task queue `brevio-main`.

## Operations

### Verify Brain Service Health
```bash
curl https://brain.brevio.app/health
```

### Check Temporal Connection
Health response includes `"temporal": "connected"` or `"unavailable"`.

### Manual Message Injection (Testing)
```bash
curl -X POST https://brain.brevio.app/v1/brain/ingest?workspace_id=<WS_ID> \
  -H "Content-Type: application/json" \
  -d '{
    "id": "<UUID>",
    "channel": "API",
    "user_id": "<USER_UUID>",
    "timestamp": "2026-01-01T00:00:00Z",
    "content": {"type": "TEXT", "text": "test message"},
    "metadata": {"channel_message_id": "<UUID>", "session_id": "<SESSION_UUID>"},
    "context": {"user_profile_hash": "<64_HEX_CHARS>"}
  }'
```

## Incident: Messages Not Processing

1. Check brain service health endpoint.
2. Verify Temporal is reachable: `TEMPORAL_HOST` env var.
3. Check Temporal UI for failed/stuck workflows.
4. Check temporal-worker logs for activity failures.
5. Verify `brevio-main` task queue has active workers.

## Incident: Duplicate Processing

Idempotency is enforced via Temporal workflow ID. If `msg-{id}` workflow already exists,
Temporal returns the existing run. No action needed.

## Rollback
Brain ingress is stateless. To stop processing:
1. Scale brain service replicas to 0.
2. Messages will queue in the caller; no data loss.
