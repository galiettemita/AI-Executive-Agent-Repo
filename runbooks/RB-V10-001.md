# RB-V10-001: Kill Switch Activation/Deactivation

## Scope
Emergency kill switch for workspace or user-level service suspension.

## Trigger Conditions
- Billing fraud detected
- Compromised workspace credentials
- Abuse or ToS violation
- Operator escalation request

## Activation Procedure

### Via Admin API
```bash
curl -X POST https://admin.brevio.app/v1/admin/kill-switch/activate \
  -H "X-User-Role: admin" \
  -H "Content-Type: application/json" \
  -d '{
    "workspace_id": "<WORKSPACE_ID>",
    "user_id": "",
    "activated_by": "<ADMIN_USER_ID>",
    "reason": "INC-XXXX: <description>"
  }'
```

### Workspace-level vs User-level
- **Workspace-level**: Leave `user_id` empty. Blocks ALL users in workspace.
- **User-level**: Set `user_id` to target user. Only blocks that user.

## Deactivation Procedure
```bash
curl -X POST https://admin.brevio.app/v1/admin/kill-switch/deactivate \
  -H "X-User-Role: admin" \
  -H "Content-Type: application/json" \
  -d '{
    "workspace_id": "<WORKSPACE_ID>",
    "user_id": ""
  }'
```

## Verification
```bash
curl https://admin.brevio.app/v1/admin/kill-switch \
  -H "X-User-Role: admin"
```

## Rollback
Deactivation is the rollback. Kill switches are stateless toggles.

## Escalation
If kill switch activation fails, contact platform on-call and manually disable workspace at the database level:
```sql
UPDATE workspaces SET status = 'suspended' WHERE id = '<WORKSPACE_ID>';
```
