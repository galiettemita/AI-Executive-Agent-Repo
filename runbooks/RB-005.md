# RB-005: Connector Outage and Circuit Breaker Activation

## Trigger
- Circuit opens for provider/workspace pair (>=5 failures in 60s).
- Elevated connector error rates and latency p95 breaches.

## Detection
- Review connector health table and recent executor failures.
- Confirm breaker state transitions (open -> half-open -> closed).

## Immediate Actions
1. Route calls to fallback provider/tool path where available.
2. Disable non-critical connector writes.
3. Notify affected workspaces with degraded-mode message.

## Mitigation
- Increase retry backoff and reduce concurrency to the provider.
- Validate OAuth token freshness and permissions.
- Coordinate with vendor status updates.

### Connector-Specific Checks (Wave 1)
- `google_calendar`: verify calendar write scope and target calendar ACL.
- `google_drive`: verify upload scope, folder permission, and file-size limits.
- `google_gmail`: verify send scope and sender/delegation policy.
- `notion`: verify workspace integration permissions and page-write access.
- `todoist`: verify project access and task-write scope.
- `brave_search`: verify API key validity and query quota.
- `github`: verify PAT scopes (`repo`, `issues`) and org access.
- `apple_reminders`: verify integration token and reminders bridge access.

## Recovery
- Close breaker only after successful half-open probe sequence.
- Restore non-critical connectors gradually.

## Verification
- Error and latency metrics return within SLO thresholds.
- No duplicate side effects from retried commits.

## Escalation
- Escalate to vendor and on-call if outage lasts > 30 minutes.
