# SOC 2 Readiness Plan

## Scope
- Trust services criteria: Security, Availability, Confidentiality
- In-scope systems: API services, Render deploys, PostgreSQL, Redis, object storage, messaging integrations, and supporting CI/CD
- Out of scope: user devices and non-production personal environments

## Tracker
- Primary tracker: docs/SOC2_READINESS_TRACKER.csv
- Status values: Planned, In Progress, Done, Needs Review
- Evidence: link to screenshots, logs, policies, or tickets (store evidence outside the repo)

## Phases
1. Baseline policies and controls (security, access control, change management, vendor management)
2. Implement controls and start evidence collection
3. Gap remediation and pre-audit readiness review
4. Type I audit, then Type II observation window (3-6 months)

## Roles
- Security Owner: overall control ownership and evidence review
- Engineering Owner: technical control implementation
- Compliance Owner: policy maintenance and auditor coordination

## Review Cadence
- Monthly tracker review
- Quarterly access reviews
- Annual policy review and updates
