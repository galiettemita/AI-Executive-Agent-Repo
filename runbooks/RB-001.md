# RB-001: Webhook Signature and Replay Incident

## Trigger
- Spike in `BREVIO.security.webhook.signature_invalid.v1` or `BREVIO.security.webhook.replay_blocked.v1`.
- Gateway webhook endpoint returns elevated `401` or `429` for trusted providers.

## Detection
- Check gateway logs filtered by `event=webhook_validation_failed`.
- Confirm recent secret rotation or provider-side signature key changes.

## Immediate Actions
1. Freeze outbound automation for affected channel (`system_autonomy_overrides` set to `D2`).
2. Validate active webhook secret version in Secrets Manager.
3. Compare expected HMAC base string and timestamp skew from recent requests.
4. Verify Redis nonce-cache health and TTL behavior (`600s`).

## Mitigation
- If provider key changed: rotate secret and deploy gateway config.
- If replay cache degraded: fail closed for write actions and restart Redis clients.
- If clock skew is high: correct node time and recheck validation window.

## Recovery
- Re-enable normal autonomy level after 30 minutes of clean signature checks.
- Replay only safe, idempotent inbound events from provider retry queue.

## Verification
- `invalid_signature_rate` returns to baseline.
- No unexpected duplicate ingress rows by `dedup_hash`.
- Audit records include incident start/end and secret version used.

## Escalation
- Escalate to security on-call if invalid signatures persist > 15 minutes.
- Escalate to provider support if payload canonicalization mismatch remains unresolved.
