# RB-V92-006: CRDT Conflict Storm
## Trigger
Conflict rate spikes beyond normal baseline for concurrent memory writes.

## Detection
Increase in `BREVIO_memory_conflict_total` and manual review queue growth.

## Immediate Actions
1. Raise manual-review threshold and pause aggressive auto-resolution.
2. Throttle high-frequency writers for affected workspaces.
3. Capture conflicting vector-clock payload samples.

## Mitigation
1. Validate vector clock ordering logic and merge strategy selection.
2. Patch deterministic tie-break rules if drift detected.
3. Replay representative conflicts in staging harness.

## Recovery
1. Drain manual conflict queue with prioritized triage.
2. Resume normal writer throughput in stages.
3. Re-enable full auto-resolution once stability is confirmed.

## Verification
1. Conflict rate normalizes to baseline.
2. Manual-review queue trend returns downward.
3. No data-loss or regression indicators in memory history.

## Escalation
Escalate to data consistency owner if unresolved conflicts continue for >1 hour or if high-severity data divergence is observed.
