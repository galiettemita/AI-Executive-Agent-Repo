3. Database Layer — Full DDL
Migration file: db/migrations/004_brevio_v101_admin_intelligence.sql
Applies after: 003_BREVIO_v92_production_hardening.sql
All tables: UUIDv7 PKs, workspace_id RLS, forward-only. 16 new tables defined below.


llm_cost_ledger
Token-level cost record for every LLM call. Written by Brain via outbox. Never queried from dashboard — only aggregated by Temporal workflows.

task_cost_rollup
One row per Temporal workflow execution. Written exclusively by CostRollupWorkflow.

connector_cost_ledger
Per-call cost for paid external connectors (e.g., Twilio, Google Maps API, data providers). Written by Executor via outbox.

user_cost_daily_rollup
Pre-aggregated daily cost per user. Written by DailyUserCostRollup workflow at 01:00 UTC. Admin dashboard reads ONLY this table — never llm_cost_ledger directly.

operator_margin_report
Daily operator P&L snapshot. Revenue from mrr_snapshots, COGS from user_cost_daily_rollup. Written by MarginSnapshotWorkflow at 02:00 UTC.


agent_kill_switches
Per-user kill switch state. ExecutionGate checks this table first, before any other gate. One row per user (upsert on activation, delete on deactivation).

skill_acl_overrides
Per-user, per-skill enable/disable overrides. Default behavior (no row) follows workspace-level feature flags. A row here overrides for the specific user.


subscription_events
Raw events from Stripe webhooks. Append-only. Source of truth for all revenue calculations.

mrr_snapshots
Daily MRR/ARR snapshot computed by MRRSnapshotWorkflow from subscription_events.

user_cohorts
Cohort assignment: every user is assigned to a signup_week cohort at registration. Retention rows are computed weekly by CohortComputeWorkflow.


oauth_token_registry

behavioral_risk_scores

feature_adoption_events

tool_mttr_log

agent_action_replay_log
