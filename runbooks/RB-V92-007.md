# RB-V92-007: Compliance DSR SLA Risk
## Trigger
Any DSR request reaches the 5-day-to-deadline warning window.

## Detection
`DSR_SLA_AT_RISK` escalation events and aging report anomalies.

## Immediate Actions
1. Assign explicit owner and backup for each at-risk request.
2. Prioritize DSR queue over non-urgent compliance tasks.
3. Notify legal/compliance contact of timeline risk.

## Mitigation
1. Validate evidence completeness and identity verification artifacts.
2. Remove process blockers (third-party export, connector callback, approvals).
3. Run daily checkpoint until request closes.

## Recovery
1. Complete request lifecycle and issue user confirmation.
2. Archive compliance evidence with hash integrity metadata.
3. Document root cause of near-SLA breach for process improvement.

## Verification
1. All at-risk DSRs move to completed before deadline.
2. No missing evidence hash or audit trail fields.
3. SLA dashboard returns to healthy state.

## Escalation
Escalate to compliance lead and legal counsel if any DSR risks missing statutory deadline.
