1. Executive Architecture Overview
Brevio is an AI executive assistant operating through WhatsApp and iMessage, functioning as a digital chief of staff. The system integrates 153 OpenClaw community skills across a 3-plane architecture (Gateway/Brain/Hands) connecting to 40+ external services to take real-world actions on behalf of users.

1.1 System Identity

1.2 Three-Plane Architecture
Every message flows through three deterministic planes in sequence. Each plane has bounded responsibilities and explicit contracts with adjacent planes.

1.2.1 Gateway Plane (Message I/O Layer)
Purpose: Ingress/egress normalization, channel-specific formatting, voice processing, rate limiting.
Skills Count: 8 skills (asr, openai-tts, gemini-stt, sag, voice-wake-say, whatsapp-styling-guide, vocal-chat, autoresponder)
Bounded Context: No business logic. No intent classification. Stateless except for session tracking. DOES call external APIs for voice processing (STT/TTS). See Section 15D for details.
Receives raw message from WhatsApp/iMessage webhook
Normalizes to internal MessageEnvelope schema
Handles voice-to-text transcription (asr, gemini-stt)
Formats outbound response per channel (whatsapp-styling-guide)
Handles text-to-speech for voice responses (openai-tts, sag)
Enforces rate limiting per user (token bucket: 60 req/min, 1000 req/hr)

1.2.2 Brain Plane (Decision & Orchestration Layer)
Purpose: Intent classification, task decomposition, skill routing, behavioral intelligence, multi-step planning.
Skills Count: 16 skills (doing-tasks, plan-my-day, daily-rhythm, morning-manifesto, personal-shopper, clawringhouse, smart-expense-tracker, card-optimizer, refund-radar, contract-reviewer, meeting-autopilot, proactive-research, focus-mode, thinking-partner, relationship-skills, self-improvement)
Bounded Context: Reads user context (USER.md, SOUL.md, AGENTS.md knowledge files). Decides which Hands skills to invoke. Does NOT call external APIs directly.
Classifies intent from normalized MessageEnvelope
Loads user behavioral profile from knowledge files
Decomposes complex requests into parallel/sequential task graphs
Routes each sub-task to appropriate Hands-plane skill(s)
Aggregates results from multiple skill executions
Generates contextual, personality-aware response

1.2.3 Hands Plane (Execution Layer)
Purpose: External service execution, API calls, real-world actions, data retrieval, state mutations.
Skills Count: 129 skills across 16 categories
Bounded Context: Executes exactly one skill per invocation. Returns structured SkillResult. No user-facing formatting. No intent classification.


1.3 Data Flow Graph
Deterministic message processing pipeline:

1. INGRESS [WhatsApp/iMessage Webhook]
   Raw message received via webhook endpoint
2. GATEWAY.normalize() [Gateway Plane]
   Channel-specific parsing, voice transcription, MessageEnvelope creation
3. BRAIN.classify() [Brain Plane]
   Intent classification, user context loading, behavioral profile merge
4. BRAIN.decompose() [Brain Plane]
   Task graph generation with dependency DAG, parallel/sequential routing
5. HANDS.execute() [Hands Plane] (per-skill, parallelized where independent)
   External API calls, data retrieval, mutations, SkillResult return
6. BRAIN.aggregate() [Brain Plane]
   Result merging, response generation, personality overlay
7. GATEWAY.format() [Gateway Plane]
   Channel-specific formatting, TTS if voice, message delivery
8. EGRESS [WhatsApp/iMessage API]
   Delivery confirmation, read receipt tracking, webhook acknowledgment

6.2.1 Intent Classification Prompt
ID: PROMPT_INTENT_CLASSIFY_V1
Model: claude-haiku-4-5 (cost-optimized)
Max Output Tokens: 256
Temperature: 0.1 (near-deterministic)

System: You are Brevio's intent classifier. Given a user message and their profile context, output a JSON object with: intent (string), confidence (0.0-1.0), skills (string[]), requires_decomposition (boolean). You must select from the closed set of registered skill IDs. Never hallucinate skill IDs.
Input Schema: { message_text: string, user_profile: { timezone, locale, enabled_skills[], recent_intents[] }, context: { time_of_day, day_of_week, active_session_minutes } }
Output Schema: { intent: string, confidence: number (0.0-1.0), skills: string[], requires_decomposition: boolean, reasoning: string }
Validation: confidence >= 0.7 required. All skill IDs must exist in skills.registry. Max 5 skills per classification.
Guardrail: If confidence < 0.7, fall back to keyword-based classifier. If output fails JSON parse, retry once with stricter prompt.

6.2.2 Response Generation Prompt
ID: PROMPT_RESPONSE_GEN_V1
Model: claude-sonnet-4-5 (quality-optimized)
Max Output Tokens: 2048
Temperature: 0.3

System: You are Brevio, an executive AI assistant communicating via {channel}. You have the user's personality profile (SOUL.md), preferences, and context. Generate a natural, concise response incorporating skill execution results. Match the user's communication style. Use {channel_formatting} rules.
Input: { skill_results: SkillResult[], user_profile: UserProfile, channel: string, conversation_history: Message[] }
Output: { response_text: string, suggested_actions: string[], follow_up_scheduled: boolean }
Guardrail: Response must not exceed 4096 chars. Must not contain PII not already known to user. Must not make financial commitments without explicit user confirmation.

6.2.3 Task Decomposition Prompt

15A.3 Edge Agent Architecture
Component: brevio-edge-agent (installed on user's Mac via DMG/Homebrew)
Runtime: Node.js 20 LTS process running as macOS Launch Agent
Communication: Persistent WebSocket connection to brevio-hands cloud service
Auth: mTLS with per-device client certificate issued during onboarding

Flow for local skills:
1. Brain routes task to skill with deployment_mode = 'local_mac'
2. brevio-hands sends skill execution request over WebSocket to user's Edge Agent
3. Edge Agent executes skill locally (AppleScript, CLI, filesystem)
4. Edge Agent returns SkillResult over WebSocket
5. brevio-hands aggregates with other cloud skill results

Offline handling: If Edge Agent is disconnected, skill execution is queued (max 4 hours). User is notified: 'I need your Mac to be online to do that.'
Security: Edge Agent runs sandboxed. No internet access except WebSocket to Brevio. All data encrypted in transit. No local skill can access data from another user.

17.1 ISkillAdapter Interface (TypeScript)
Every skill MUST implement this interface. No exceptions. No stubs.

interface ISkillAdapter {
  readonly id: string;                    // Must match skills.registry.id
  readonly plane: 'gateway' | 'brain' | 'hands';
  readonly requiredScopes: string[];      // OAuth scopes needed
  readonly inputSchema: JSONSchema7;      // Validated before execute()
  readonly outputSchema: JSONSchema7;     // Validated after execute()
  execute(input: SkillInput, ctx: SkillContext): Promise<SkillResult>;
  healthCheck(): Promise<boolean>;        // Called by circuit breaker probe
  undo?(input: SkillInput): Promise<void>; // Optional compensating action
}
