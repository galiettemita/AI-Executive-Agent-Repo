# RB-V92-005: Feature Flag Rollback
## Trigger
Error or latency spike is correlated with recent feature-flag change.

## Detection
Change audit trail shows recent flag toggle preceding metric regression.

## Immediate Actions
1. Disable suspect flag(s) for all tenants (kill switch).
2. Snapshot flag evaluation and rollout cohort state.
3. Notify stakeholders that rollback is in progress.

## Mitigation
1. Narrow blast radius to canary cohort for repro.
2. Inspect rule targeting and precedence ordering.
3. Patch flag evaluation bug or invalid rule definition.

## Recovery
1. Re-enable feature in staged cohorts after fix validation.
2. Verify no hidden dependency on previous rollout state.
3. Update flag rollout checklist with incident learnings.

## Verification
1. Error/latency metrics return to baseline.
2. No new alerts tied to the rolled-back feature.
3. Audit log confirms expected flag states across environments.

## Escalation
Escalate to release manager if rollback fails or multiple flags are suspected as interacting causes.
