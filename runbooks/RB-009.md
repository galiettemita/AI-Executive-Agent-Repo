# RB-009: DR and Restore Drill

## Trigger
- Scheduled monthly restore drill or declared disaster event.
- RDS/PITR recovery objective test window begins.

## Detection
- Confirm latest snapshots and backup retention policy.
- Validate Terraform/Helm manifests for recovery environment.

## Immediate Actions
1. Launch isolated restore environment from latest snapshot/PITR point.
2. Restore critical services in dependency order: DB -> cache -> queues -> control-plane services.
3. Validate secrets and key versions before enabling traffic.

## Mitigation
- If restore fails, fall back to previous known-good restore point.
- Resolve schema/version mismatch with forward-only migration patch.

## Recovery
- Execute smoke tests for gateway/control/executor/workflows.
- Record actual RTO/RPO values and gaps.

## Verification
- Core message pipeline and critical workflows pass in restored environment.
- Audit log chain and workspace isolation checks pass post-restore.

## Escalation
- Escalate to incident commander if RTO/RPO targets are missed.
