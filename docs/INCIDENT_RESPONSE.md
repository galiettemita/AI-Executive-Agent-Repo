# Incident Response

## Severity Levels
- P0: Complete outage, data loss, or security incident
- P1: Major feature degraded or widespread errors
- P2: Partial degradation, limited scope
- P3: Minor issue with workaround

## Response Steps
1. Acknowledge and assign incident owner
2. Establish communication channel
3. Mitigate impact, then identify root cause
4. Create and track follow-up actions
5. Postmortem within 5 business days

## Communication
- Status page update every 30–60 minutes during P0/P1
- Internal updates in #incident channel
