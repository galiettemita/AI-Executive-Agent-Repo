# RB-V92-008: Guardrail False Positive Storm
## Trigger
Guardrail block rate exceeds 5% with confirmed benign traffic.

## Detection
`BREVIO_guardrail_block_total` spike and operator triage of blocked safe requests.

## Immediate Actions
1. Shift selected guardrails from block to warn mode where policy allows.
2. Capture blocked payload samples for rule-level analysis.
3. Inform support/ops of temporary guardrail posture adjustment.

## Mitigation
1. Tune thresholds and rule weighting for offending classifiers.
2. Deploy targeted allowlist for confirmed safe patterns.
3. Run prompt-injection regression suite before re-tightening rules.

## Recovery
1. Return guardrails to intended enforcement mode incrementally.
2. Purge temporary allowlists no longer needed.
3. Publish postmortem with precision/recall impact summary.

## Verification
1. False-positive rate returns to target range.
2. No increase in true-positive misses for critical attacks.
3. Regression suite passes with updated rules.

## Escalation
Escalate to security lead when false positives remain high after threshold tuning or if policy exceptions become broad.
